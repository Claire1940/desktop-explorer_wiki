En la aventura de terror digital retro de los 90, Desktop Explorer, los jugadores deben navegar por un entorno simulado de Windows 95 para resolver un oscuro misterio que se va revelando. El sistema operativo simulado esconde pruebas críticas detrás de directorios restringidos, sectores dañados y archivos protegidos por contraseña dejados en la vieja computadora de un tío.

El sistema de archivos virtual está estructurado de manera similar a los sistemas operativos antiguos, lo que requiere que los jugadores utilicen directorios de comandos básicos, aplicaciones de diagnóstico y herramientas de recuperación de archivos. <Section icon={Sparkles} title="Cómo Descubrir y Restaurar Archivos de Sistema Eliminados">

Para avanzar en la historia, los jugadores deben aprender a recuperar datos eliminados del disco duro simulado. Estos archivos eliminados a menudo contienen registros de chat fragmentados, correos electrónicos faltantes y configuraciones del sistema que desbloquean software restringido.

Restaurar estos archivos requiere utilizar la Papelera de reciclaje integrada, ejecutar parches del registro y ejecutar herramientas simples de línea de comandos. Muchos de estos archivos están deliberadamente fragmentados, lo que requiere encontrar múltiples piezas antes de que puedan ser leídos.

<Steps>
  <Step num="1" title="Inspeccionar la Papelera de Reciclaje">
    Abre la Papelera de reciclaje del escritorio para encontrar fragmentos de texto básicos. Algunos archivos estarán bloqueados o requerirán una utilidad del sistema para restaurarlos a su directorio original.
  </Step>
  <Step num="2" title="Localizar la Utilidad del Editor Hexadecimal">
    Busca en la carpeta `/System/` la aplicación del editor hexadecimal. Esta utilidad es necesaria para reparar los encabezados de archivos que han sido dañados o modificados intencionalmente.
  </Step>
  <Step num="3" title="Aplicar Parches del Registro">
    Accede al editor de registro virtual para cambiar los permisos. Cambiar los valores del directorio de 0 a 1 hace que las carpetas ocultas sean visibles en el escritorio principal.
  </Step>
  <Step num="4" title="Ejecutar Comando de Recuperación">
    Ejecuta la utilidad de reparación en el archivo dañado seleccionado. Una vez restaurado por completo, el documento se puede abrir utilizando el visor de texto retro predeterminado.
  </Step>
</Steps>

| Archivo Eliminado | Método de Recuperación | Impacto en la Historia |
| :--- | :--- | :--- |
| boot_log.bak | Restauración de Papelera | Explica el fallo inicial del sistema |
| chat_09.txt | Utilidad de editor hexadecimal | Revela la conversación con el Tío |
| config.sys | Ejecución de parche de registro | Desbloquea el subdirectorio oculto |

<Callout type="warning" title="Riesgos de Alteración del Registro">
Modificar los valores del registro sin localizar las pistas adecuadas en el juego puede bloquear ciertos directorios de forma permanente para esa partida. Siempre coteja las notas adhesivas antes de editar valores hexadecimales.
</Callout>

Una vez que estos archivos se recuperan, a menudo apuntan directamente a la siguiente fase de la investigación, proporcionando nombres, fechas y credenciales de cuenta que antes faltaban.

</Section>

Muchos **archivos ocultos de Desktop Explorer** están bloqueados tras contraseñas complejas que requieren razonamiento deductivo para resolverse. El juego no utiliza códigos aleatorios; en su lugar, cada contraseña está oculta contextualmente dentro de otros archivos, configuraciones del sistema o elementos retro del escritorio.

Para descifrar estas carpetas, los jugadores deben examinar el entorno del escritorio de manera integral, buscando pistas en lugares inesperados como fechas de creación de archivos, registros de errores del sistema o metadatos ocultos.
