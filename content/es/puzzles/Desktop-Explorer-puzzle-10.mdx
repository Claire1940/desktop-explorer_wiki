<Section icon={Map} title="Pistas necesarias y ubicaciones de directorios">

Para resolver con éxito el **acertijo 10 de Desktop Explorer**, debes reunir tres pistas fragmentadas dispersas por el sistema operativo virtual. La entidad que corrompe la computadora ha escondido estas piezas dentro de archivos del sistema, carpetas eliminadas y antiguos registros de correo electrónico.

Navegar por la estructura de directorios requiere una atención estricta a los detalles, ya que muchos de estos archivos están marcados con atributos ocultos. A continuación se presenta el desglose preciso de las ubicaciones y archivos que necesitas asegurar antes de intentar el proceso de descifrado.

| Nombre del archivo | Ruta del directorio | Tipo de pista | Propósito |
|--- |--- |--- |--- |
| RECOVERY_10.TMP | C:\RECYCLE_BIN | Hex fragmentado | Archivo base para descifrado |
| MAIL_LOG_07.TXT | C:\MAIL\ARCHIVE | Valor de desplazamiento | Clave de cifrado César |
| REG_KEY_P10.DAT | C:\WINDOWS\SYSTEM | Hex de registro | Firma de validación |

<Section icon={Target} title="Guía paso a paso para el acertijo 10 de Desktop Explorer">

Una vez que hayas reunido todos los archivos y pistas necesarios, puedes proceder con la secuencia de descifrado. Este proceso debe ejecutarse dentro de la Terminal virtual para evitar que el sistema operativo se bloquee debido a la corrupción del sistema de archivos.

Sigue estos pasos cuidadosamente para reconstruir el bloque de datos corrupto y desbloquear los registros finales del tío.

<Steps>
  <Step num="1" title="Acceder al símbolo del sistema">
    Inicia el programa Terminal virtual desde el Menú de Inicio. Navega directamente al directorio de recuperación del sistema ingresando los comandos de recorrido de directorio.
  </Step>
  <Step num="2" title="Reconstruir el archivo temporal">
    Mueve el archivo `RECOVERY_10.TMP` recuperado desde el escritorio al directorio de recuperación activo. Usa la línea de comandos del sistema de archivos para verificar la presencia del archivo.
  </Step>
  <Step num="3" title="Aplicar el cifrado hexadecimal">
    Abre la utilidad Hex Editor. Carga el archivo temporal y aplica el desplazamiento de cifrado César +4 a los encabezados del archivo para reparar la secuencia binaria corrupta.
  </Step>
  <Step num="4" title="Ejecutar el parche del registro">
    Abre el Editor del Registro virtual. Localiza la carpeta de validación e inyecta la firma hexadecimal reparada para desbloquear permanentemente los registros restringidos del sistema.
  </Step>
</Steps>

| Comando | Sintaxis | Resultado esperado |
|--- |--- |--- |
| CD | cd C:\RECOVERY\TEMP10 | Navega al directorio |
| DIR /A | dir /a | Muestra archivos ocultos |
| DECRYPT | decrypt RECOVERY_10.TMP | Abre la herramienta de descifrado |
| REGEDIT | regedit | Inicia el editor del registro |

<Callout type="warning" title="Advertencia de fallo (Glitch)">
Ejecutar el comando de descifrado sin aplicar el desplazamiento hexadecimal exacto de +4 activa un evento de fallo del sistema, forzando un reinicio completo del sistema virtual.
</Callout>

</Section>

<Section icon={Shield} title="Solución de problemas y mitigación de fallos">

Durante el proceso de descifrado del **acertijo 10 de Desktop Explorer**, el sistema operativo virtual puede mostrar un comportamiento inestable. Esta es una mecánica de terror intencional diseñada para desorientarte. La pantalla CRT parpadeará, se reproducirán archivos de audio distorsionados de fondo y aparecerán avisos de error falsos en el escritorio.

Si encuentras un código de error durante la ejecución, consulta la tabla a continuación para resolver los bloqueos del sistema de inmediato.

| Código de error | Significado | Resolución |
|--- |--- |--- |
| ERR_10_BAD_HEADER | Encabezado de archivo corrupto | Vuelve a descargar el archivo de recuperación |
| ERR_10_KEY_MISMATCH | Valor de desplazamiento incorrecto | Vuelve a leer el archivo de correo |
| ERR_10_REG_LOCK | Registro protegido contra escritura | Ejecuta la Terminal como administrador |

<Callout type="success" title="Bypass de Modo Seguro">
Si el escritorio virtual entra en un bucle de bloqueo infinito, mantén presionada la tecla Shift durante la secuencia de inicio virtual para forzar al sistema a entrar en Modo Seguro.
</Callout>

</Section>
