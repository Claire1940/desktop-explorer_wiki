在执行解密序列之前，你必须收集隐藏在计算机文件系统中的三个不同的数据碎片。这些碎片提供了读取 `SYS_LOG_06.DAT` 内加密数据所需的精确二进制偏移量。

操作系统将这些文件隐藏在标准系统文件夹下以防止意外删除，这模仿了现实世界中 Windows 95 的目录结构。你需要切换隐藏文件的可见性，并探索乍一看似乎空无一物的目录。

<CardGrid cols={3}>
  <Card title="碎片 1：CONFIG.SYS">
    - **位置**：根目录 `C:\`
    - **类型**：系统配置文件
    - **线索**：包含对齐终端内存地址块所需的初始十六进制偏移量乘数。
  </Card>
  <Card title="碎片 2：MEM_DUMP.BMP">
    - **位置**：临时文件夹 `C:\WINDOWS\TEMP\`
    - **类型**：隐藏图像文件
    - **线索**：显示高亮像素的视觉网格，直接映射到目标文件的十六进制坐标系。
  </Card>
  <Card title="碎片 3：TRASH_09.MSG">
    - **位置**：邮件数据库 `C:\WINDOWS\MAIL\TRASH\`
    - **类型**：已删除的邮件消息
    - **线索**：一封恢复的电子邮件，包含完成主密码输入所需的字母数字后缀。
  </Card>
</CardGrid>

<Section icon={Swords} title="Desktop Explorer 谜题 6 逐步解决方案">

一旦你从上述目录中收集了所有三个线索，就可以开始解密过程了。请仔细按照以下步骤重构目录并输入正确的序列。

<Steps>
  <Step num="1" title="初始化虚拟终端">
    双击桌面上的 MS-DOS 提示符图标。输入 `CD WINDOWS\SYSTEM` 并按回车键以导航到系统子目录。从那里，输入 `CD RECOVERY` 进入包含目标文件的文件夹。
  </Step>
  <Step num="2" title="执行目录重构">
    在终端中输入命令 `DEFRAG /SYS /FORCE`。该命令会重新对齐虚拟硬盘的碎片化扇区，使系统能够读取 `SYS_LOG_06.DAT` 的损坏块而不会崩溃。
  </Step>
  <Step num="3" title="提取十六进制偏移量">
    从开始菜单中打开虚拟十六进制编辑器应用程序。将 `SYS_LOG_06.DAT` 加载到编辑器中。将 `MEM_DUMP.BMP` 中高亮的网格坐标与十六进制行进行交叉比对。记下偏移量 `0x0F`、`0x4A` 和 `0x92` 处的值。
  </Step>
  <Step num="4" title="编译并输入主密码">
    将提取的十六进制值与在 `TRASH_09.MSG` 中找到的字母数字后缀相结合。最终编译的密钥格式如下：`[Hex1][Hex2]-[Suffix]`。在运行 `RECOVERY.EXE` 时出现的安全对话框中输入此密钥。
  </Step>
</Steps>

<Callout type="success" title="解密成功">
输入正确的密码后，终端屏幕将闪烁绿光，并且在你的根目录下会生成一个名为 `UNLOCKED_LOG.TXT` 的新文本文件，从而推进游戏剧情。
</Callout>

下表概述了在此谜题序列期间成功导航终端所需的精确命令和操作。

| 命令行输入 | 目标目录 | 预期输出 | 目的 |
|------|------|------|------|
| CD WINDOWS\SYSTEM\RECOVERY | C:\ | 提示符更改为目标文件夹 | 导航到恢复目录 |
| DIR /A:H | C:\WINDOWS\SYSTEM\RECOVERY\ | 列出目录中的隐藏文件 | 验证恢复工具是否存在 |
| DEFRAG /SYS /FORCE | C:\WINDOWS\SYSTEM\RECOVERY\ | 扇区对齐动画 | 重构损坏的系统块 |
| RUN RECOVERY.EXE | C:\WINDOWS\SYSTEM\RECOVERY\ | 出现密码输入提示 | 启动解密界面 |

</Section>

<Section icon={Shield} title="故障排除与错误代码">

由于 Desktop Explorer 模拟的是一个旧操作系统，如果输入命令的顺序不对或文件访问不正确，玩家经常会遇到逼真的系统错误。了解如何解决这些错误对于完成谜题至关重要。

如果你在十六进制对齐过程中犯了错误，虚拟操作系统将显示蓝屏或抛出文件访问错误。不要慌张；系统设计允许快速恢复，而不会丢失你的整体游戏进度。

<Callout type="warning" title="避免终端软锁死">
如果你在 `RECOVERY.EXE` 程序中输入错误密码三次，终端将被锁死。要绕过此限制，你必须从根目录运行命令 `SYS_RESET.BAT` 以清除锁定状态。
</Callout>

| 错误代码 | 潜在原因 | 解决所需的操作 |
|------|------|------|
| ERROR 404: FILE NOT FOUND | 使用正斜杠输入目录路径 | 使用反斜杠 (\\) 进行 Win95 目录导航 |
| ERROR 0x09: CORRUPT BLOCK | 在运行 DEFRAG 命令之前打开十六进制编辑器 | 关闭编辑器，运行 DEFRAG /SYS /FORCE，重新打开编辑器 |
| ERROR 0xF1: ACCESS DENIED | 在终端打开时尝试编辑 CONFIG.SYS | 在修改系统文件之前关闭 MS-DOS 提示符 |
| BSOD: FATAL EXCEPTION 0E | 直接从资源管理器双击 DAT 文件 | 仅通过十六进制编辑器应用程序打开 DAT 文件 |

</Section>
