<Section icon={Terminal} title="一步步解密通关步骤">

请按照以下精确步骤绕过 Archive_08.dat 的安全保护。此步骤序列假定你已还原了命令提示符并收集了上一节中提到的必要文件。

<Steps>
  <Step num="1" title="初始化终端">
    从桌面打开命令提示符。输入 `cd C:\WINDOWS\SYSTEM\TEMP` 并按回车。你应该会看到目录发生改变。如果找不到该目录，说明你没有在文件资源管理器的“查看”菜单中启用“显示隐藏文件”。
  </Step>
  <Step num="2" title="确定偏移量">
    在记事本程序中打开 “Mail_Log.txt”。找到来自“Uncle”的最后一条记录。注意时间戳（通常是晚上 11:42）。该谜题的“偏移量”是小时与分钟之间的差值（42 - 11 = 31）。
  </Step>
  <Step num="3" title="提取二进制字符串">
    打开 “System.ini” 并找到以 `A8_KEY=` 开头的行。你会看到一串 8 位数的数字。将你的偏移量（31）与该字符串的前两位数字相加。
  </Step>
  <Step num="4" title="执行解密命令">
    返回命令提示符。输入命令 `DECRYPT -f Archive_08.dat -k [你的计算密钥]`。将括号内的文本替换为你最终计算出的 8 位数字序列。
  </Step>
  <Step num="5" title="获取内容">
    一旦出现 “Decryption Successful”（解密成功）消息，桌面上就会出现一个名为 “RECOVERED_DATA” 的新文件夹。打开它即可完成 Desktop Explorer 谜题 8。
  </Step>
</Steps>

| 命令语法 | 功能 | 预期输出 |
| :--- | :--- | :--- |
| **DIR /H** | 显示当前文件夹中的隐藏文件 | 包含 Archive_08.dat 的列表 |
| **HELP DECRYPT** | 显示命令参数 | 用法: DECRYPT -f [文件] -k [密钥] |
| **CLS** | 清除终端屏幕 | 空白终端窗口 |
| **TYPE System.ini** | 在终端中显示文件内容 | 系统设置的文本输出 |

<Section icon={Shield} title="常见错误排查">

由于游戏严格模拟了 90 年代操作系统的局限性，许多玩家在进行 Desktop Explorer 谜题 8 时会遇到问题。如果你的命令失效或文件无法打开，请参考下表寻找常见解决方法。

| 错误消息 | 可能原因 | 推荐解决方案 |
| :--- | :--- | :--- |
| **Access Denied（拒绝访问）** | 文件被标记为“只读” | 右键点击文件，取消勾选“只读” |
| **Invalid Key（无效密钥）** | 步骤 2 中的计算错误 | 重新检查 Mail_Log 的时间戳 |
| **Stack Overflow（栈溢出）** | 打开的窗口过多 | 在运行 CMD 之前关闭记事本和画图工具 |
| **File Not Found（找不到文件）** | 目录路径不正确 | 确保你处于 \SYSTEM\TEMP 目录下 |

<Callout type="warning" title="蓝屏事件">
如果你在解密过程中收到 “Fatal Exception”（致命异常）错误，请不要手动重启游戏。这是 Desktop Explorer 谜题 8 中的一个剧情杀（脚本事件）。等待模拟电脑“重启”，你会在桌面上发现一个名为 “RECOVERY.txt” 的新提示文件，它将提供密钥的前四位数字。
</Callout>
