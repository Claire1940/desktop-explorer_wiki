<Section icon={Map} title="Desktop Explorer PC 新手指南">

探索 Desktop Explorer PC 的模拟操作系统需要一种条理清晰的数字调查方法。游戏将你置于一个模仿 90 年代中期计算时代的、带有怀旧 CRT 滤镜的桌面界面前。你的主要目标是通过挖掘被遗弃工作站留下的文件、日志和应用程序，调查一起失踪案。

要开始调查，你必须熟悉桌面界面的核心交互元素。与现代操作系统不同，这种复古环境严重依赖于目录导航、手动文件关联和命令行执行。

| 应用名称 | 主要功能 | 主要线索位置 |
| :--- | :--- | :--- |
| 文件资源管理器 (File Explorer) | 浏览目录 | 隐藏的子文件夹 |
| 聊天日志 95 (ChatLog 95) | 阅读存档消息 | 缺失的时间戳 |
| 回收站 (Trash Bin) | 检索已删除的文件 | 损坏的文档碎片 |
| 网络浏览器 (NetBrowser) | 访问本地内网 | 存档的新闻文章 |

理解这些程序如何交互至关重要。例如，在回收站中发现的损坏文档可能包含解锁文件资源管理器中安全目录所需的部分密码。有关游戏更新和公告的更多详细信息，请查看官方 [Desktop Explorer Linktree](https://linktr.ee/desktop_explorer)。

<Callout type="tip" title="调查技巧">
务必检查文件的属性。有时，文件创建日期或文件大小可以揭示文档是否被修改过，或者文件末尾是否附加了隐藏数据。
</Callout>

<Section icon={Target} title="系统流程分步指南">

完成 Desktop Explorer PC 的主线叙事涉及解决连续的安全谜题并拼凑支离破碎的日志。请使用此分步进度指南，系统地解开系统的秘密。

<Steps>
  <Step num="1" title="初始系统启动与发现">
    启动虚拟机并分析默认桌面布局。打开桌面上的 ReadMe.txt 文件以了解即时背景。你的第一个目标是找到用户配置文件文件夹，该文件夹目前被锁定在一个基础密码界面后。
  </Step>
  <Step num="2" title="文件恢复与密码提取">
    导航至回收站并还原已删除的文本文件。其中一个文件包含一系列日期。将这些日期与日历应用程序进行交叉引用，以找到正确的管理员密码。输入此密码以解锁主要用户目录。
  </Step>
  <Step num="3" title="网络日志分析">
    打开 NetBrowser 应用程序并查看离线缓存。你会发现用户与一名匿名联系人之间的存档消息线程。仔细观察文本中嵌入的高亮坐标和系统错误代码。
  </Step>
  <Step num="4" title="安全解密与系统清除">
    在系统目录中找到安全分区可执行文件 (.EXE)。使用解密工具解码最终存档。此操作将触发一系列奇怪的系统事件，直接导向剧情的高潮。
  </Step>
</Steps>

| 阶段 | 主要目标 | 关键寻找物品 |
| :--- | :--- | :--- |
| 第一阶段 | 访问受限目录 | 管理员密码注释 |
| 第二阶段 | 解密私人聊天记录 | 解密密钥文件 |
| 第三阶段 | 恢复损坏的系统文件 | 文件修复可执行文件 |
| 第四阶段 | 执行系统清除 | 最终覆盖代码 |

遵循这一结构化路径可确保你不会错过解锁真结局所需的关键叙事细节。

<Callout type="warning" title="关键警告">
系统发生故障（Glitches）时请勿强制关闭应用程序。这些视觉扭曲是叙事的一部分，手动关闭程序可能会损坏你当前的存档状态。
</Callout>

</Section>

系统上的大多数文件都受到不同级别的安全保护。了解文件扩展名和所需的权限级别将为你节省数小时漫无目的的点击。

| 文件类型 | 加密级别 | 所需访问工具 |
| :--- | :--- | :--- |
| .TXT 文档 | 无 / 标准 | 记事本程序 (Notepad) |
| .LOG 系统记录 | 低 / 损坏 | 日志查看工具 (Log Viewer) |
| .DAT 加密文件 | 中 / 安全 | 解密工具 (Decryption Tool) |
| .EXE 可执行文件 | 高 / 受限 | 管理员授权 |

尝试解密 .DAT 文件时，请确保你已从 ChatLog 95 存档中提取了所有相应的密钥。

<Callout type="success" title="解密成功">
一旦你从系统目录中获得解密工具，就可以通过将文件拖放到工具图标上，直接对任何锁定的 .DAT 文件运行它。
</Callout>

<Section icon={Trophy} title="剧情结局与叙事结果">

Desktop Explorer PC 根据你的调查深度以及在最终系统清除期间做出的决定，设有多个不同的结局。仅仅完成基础任务并不能让你获得叙事的完整全貌。

| 结局 | 要求 | 结果 |
| :--- | :--- | :--- |
| 主要结局 (Main Conclusion) | 完成主要的系统修复和清除 | 解决了基础的失踪人口案件 |
| 隐藏的真相 (Hidden Truth) | 恢复全部 12 个已删除的存档碎片 | 揭示了电脑所有者黑暗的背景 |
| 系统崩溃 (System Crash) | 反复触发关键注册表错误 | 强制模拟操作系统蓝屏并损坏 |

为了达成“隐藏的真相”结局，你必须在执行系统最终命令之前，细致地编目每一封已删除的电子邮件和聊天记录碎片。

<Callout type="tip" title="结局技巧">
在执行最终系统清除之前保留一个单独的存档。这允许你重新加载并寻找任何遗漏的文件碎片，而无需重新开始整个游戏。
</Callout>

</Section>

<Section icon={Shield} title="PC 系统要求与优化策略">

由于 Desktop Explorer PC 模拟的是复古操作系统，它在各种现代电脑硬件上都能高效运行。然而，确保你的设置经过优化可以防止输入延迟和 CRT 滤镜的视觉问题。

| 类别 | 最低配置 | 推荐配置 |
| :--- | :--- | :--- |
| 操作系统 | Windows 10 64位 | Windows 11 64位 |
| 处理器 | Intel Core i3 | Intel Core i5 |
| 内存 | 4 GB RAM | 8 GB RAM |
| 存储空间 | 2 GB 可用空间 | 2 GB 可用空间 |

如果你遇到掉帧或卡顿，请尝试在游戏设置菜单中禁用高级 VHS 噪声滤镜。这可以在保持真实复古美感的同时减少处理负载。

<Callout type="warning" title="性能警告">
在超宽显示器上以真全屏模式运行游戏有时会扭曲复古的 4:3 宽高比。建议使用窗口模式以获得最真实的视觉体验。
</Callout>
