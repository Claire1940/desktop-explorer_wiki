<Section icon={Swords} title="Demo 分步通关流程">

要成功完成 Demo，玩家必须遵循特定的文件发现和系统操作顺序。谜题之间环环相扣，这意味着一个系统锁的解决方案通常可以在另一个文件的元数据中找到。

<Steps>
  <Step num="1" title="启动系统并打开记事本">
    首先探索默认的桌面布局。立即打开记事本应用程序以准备记录笔记。点击可用的文件夹目录以熟悉布局。
  </Step>
  <Step num="2" title="检查家族照片并查看元数据">
    打开位于家族文件夹中的图像文件。使用检查工具查看每张照片的元数据。密切注意作者字段和创建日期，因为这些稍后将用作凭据。
  </Step>
  <Step num="3" title="定位并拉伸壁画图像">
    找到壁画图像文件。打开它并拖动窗口边缘以拉伸图像。此操作会显现壁画边缘上隐藏的颜色序列。
  </Step>
  <Step num="4" title="运行变色应用程序">
    启动变色应用程序。将应用程序窗口与壁画并排对齐。实时更改当前颜色，使其与拉伸图像边缘中发现的序列相匹配，以解锁下一个目录。
  </Step>
  <Step num="5" title="访问祖父的日志文件">
    打开新解锁的包含您祖父日记账目的文件夹。阅读逐渐损坏的文件，以找出完成 Demo 流程所需的最终密码。
  </Step>
</Steps>

| 谜题阶段 | 所需操作 | 预期结果 |
| - | - | - |
| 初始访问 | 浏览目录 | 定位家族照片 |
| 元数据检查 | 检查作者字段 | 获取登录凭据 |
| 壁画谜题 | 拉伸图像边缘 | 显现隐藏的颜色代码 |
| 颜色对齐 | 运行变色应用 | 解锁祖父的文件夹 |
| 日记解密 | 阅读逐渐损坏的笔记 | 获取最终 Demo 密码 |

<Callout type="warning" title="逐渐损坏的文件">
随着逐条日志的推进，祖父的日记账目变得越来越混乱且难以阅读。在移动到下一个文件之前，请立即记下关键短语。
</Callout>

</Section>

<Section icon={Shield} title="平台性能与设置">

Demo 已在多种配置下进行了测试，以确保稳定的性能。由于游戏采用了模拟的操作系统的界面，它所需的系统资源极少，因此与便携式硬件高度兼容。

在 Steam Deck 上，游戏在标准控制器配置下运行流畅。触摸板可用于模拟鼠标移动，这对于界面的点击元素来说非常自然。打字谜题需要虚拟键盘，不过对于桌面玩家，物理键盘支持也已完全集成。

| 平台 | 控制输入 | 性能状态 |
| - | - | - |
| Steam Deck | 触摸板 / 触摸屏 | 经验证稳定，运行完美 |
| Windows PC | 鼠标和键盘 | 原生支持，最佳体验 |
| Linux PC | 鼠标和键盘 | 通过 Proton 完全兼容 |

音频设置允许玩家自定义他们的体验。如果复杂的解谜过程中动态音乐过渡分散了注意力，可以通过游戏内设置面板完全静音音频，而不会影响游戏机制或谜题提示。
