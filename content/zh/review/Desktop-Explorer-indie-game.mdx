<Section icon={Target} title="解密密码与解决基础谜题">

在**独立游戏 Desktop Explorer**中，通关完全取决于你定位和解密密码的能力。这些密码并非随机生成，而是隐藏在图片的元数据、已删除邮件的文本中，或混淆在系统日志内部。 

为了系统地破解这些安全措施，玩家必须学会仔细观察虚拟环境。下表详细列出了你必须访问的主要目录、在哪里可以找到它们的线索，以及所需的通用解密方法。

| 目录 | 线索来源 | 解密方法 | 难度 |
| :--- | :--- | :--- | :--- |
| **Root/Documents** | Readme.txt 提示 | 简单密码 | ★★☆☆☆ |
| **System32/Drivers** | 日志十六进制转储 | 二进制转换 | ★★★☆☆ |
| **User/Downloads** | 损坏的图像文件 | 元数据检查 | ★★★★☆ |
| **Network/Shared** | 数据包捕获日志 | 端口映射 | ★★★★★ |

要解锁你的第一个受密码保护的文件夹并开启旅程，请按照以下步骤解密序列进行操作。

<Steps>
  <Step num="1" title="定位 Readme 文件">
    导航到 `C:/User/Documents` 文件夹。打开 `Readme.txt` file。注意文本中异常分散的大写字母。
  </Step>
  <Step num="2" title="提取密码密钥">
    按顺序写下大写字母。该序列构成一个简单的凯撒密码移位密钥（通常根据你的游戏种子，移位为 +3 或 -3）。
  </Step>
  <Step num="3" title="检查图像元数据">
    右键单击同目录中名为 `hint.bmp` 的图像文件并选择“属性”。在元数据选项卡下，找到注释部分中的加密字符串。
  </Step>
  <Step num="4" title="应用移位并解锁">
    将你的凯撒密码密钥应用于元数据字符串。将生成的明文密码输入到锁定的文件夹提示框中，以获取对下一级目录的访问权限。
  </Step>
</Steps>

<Callout type="tip" title="隐藏的文件扩展名">
在游戏开始后，务必立即在系统设置面板中启用“显示隐藏文件”。几个关键的谜题线索位于隐藏的 `.ini` 和 `.sys` 文件中，这些文件在默认查看设置下不会渲染。
</Callout>

</Section>

在使用终端时，执行错误的命令可能会触发防御性安全协议，从而将你锁定在目录之外或加速异常恐怖事件的发生。请参考以下命令参考表以确保安全执行。

| 命令 | 目标实用程序 | 效果 | 威胁级别 |
| :--- | :--- | :--- | :--- |
| `sys_decrypt` | 密码文件夹 | 提示输入文件夹密钥 | 安全 |
| `scan_sector` | 硬盘扇区 | 发现损坏的文件 | 低 |
| `override_reg` | 注册表数据库 | 修改系统变量 | 高 |
| `net_connect` | 远程服务器 | 触发观察者活动 | 极高 |

<Callout type="warning" title="观察者协议">
运行 `net_connect` 命令会启动一个模拟的远程连接。这将触发“观察者（Watcher）”实体，导致你的虚拟屏幕闪烁、鼠标光标漂移以及背景音频失真。请仅在准备好所有其他线索时才执行此命令。
</Callout>
